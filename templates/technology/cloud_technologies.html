Cloud Technologies (AWS)

Our project leverages several key AWS services to ensure scalability, reliability, and security. Here's an overview of how we use AWS in our infrastructure.

EC2 (Elastic Compute Cloud)

We use EC2 to host our Django application, providing flexible compute capacity in the cloud. This allows us to easily scale our application as needed.

  • Hosts our Django application
  • Provides scalable computing capacity
  • Integrates with other AWS services

RDS (Relational Database Service)

Our PostgreSQL database is hosted on AWS RDS, which simplifies database administration tasks and provides high availability.

  • Hosts our PostgreSQL database
  • Automated backups and patch management
  • Scalable and highly available

S3 (Simple Storage Service)

We use S3 for storing and serving static files and user-uploaded content, ensuring fast and reliable access to these resources.

  • Stores static files and user uploads
  • Highly durable and available storage
  • Integrates well with CloudFront for content delivery

IAM (Identity and Access Management)

IAM helps us manage access to AWS services and resources securely. We use it to create and manage AWS users and groups, and control their access.

  • Manages access to AWS services
  • Implements principle of least privilege
  • Enhances security of our AWS infrastructure

How These AWS Services Work Together

Our EC2 instance hosts the Django application, which connects to the RDS PostgreSQL database for data storage. Static files and user uploads are stored in S3 buckets, which our Django app accesses as needed. IAM ensures that our EC2 instance and application have the necessary permissions to interact with RDS and S3, while restricting access to maintain security. This combination of services provides a robust, scalable, and secure infrastructure for our web application.
