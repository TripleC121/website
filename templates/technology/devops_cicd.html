DevOps & Deployment

Our project employs modern DevOps practices and tools to ensure smooth development, testing, and deployment processes. Here's an overview of our DevOps stack and deployment pipeline.

Docker

We use Docker for containerization, ensuring consistency across development, testing, and production environments.

  • Containers for Django app and Nginx
  • Consistent environments across stages
  • Easy scaling and deployment

GitHub Actions

Our CI/CD pipeline is implemented using GitHub Actions, automating our build, test, and deployment processes.

  • Automated testing on pull requests
  • Automated deployments to production
  • Continuous integration ensuring code quality

Nginx

Nginx serves as our web server and reverse proxy, handling incoming requests and serving static files efficiently.

  • Reverse proxy for Django application
  • Efficient serving of static files
  • SSL/TLS termination

AWS Systems Manager

We use AWS Systems Manager for secure, automated deployments to our EC2 instances.

  • Secure, remote execution of deployment scripts
  • Integration with GitHub Actions for CI/CD
  • Automated application updates

How These Technologies Work Together

Our application and its dependencies are containerized using Docker, ensuring consistency across environments. GitHub Actions automates our CI/CD pipeline, running tests on pull requests and triggering deployments when changes are merged to the main branch. The deployment process uses AWS Systems Manager to securely execute deployment scripts on our EC2 instances, updating the Docker containers with the latest code. Nginx, also running in a Docker container, acts as a reverse proxy, directing traffic to the Django application container and serving static files. This integrated approach allows for rapid, reliable, and secure deployments of our web application.
